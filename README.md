# 📡 prenos (プレノス)

**AIチェック機能付き・対話型 Nostr 投稿専門クライアント**

`prenos` は、Nostrへ投稿する前に、AIがユーザー指定のチェック観点（誤字脱字、言葉遣いの揺れ、特定のミス変換など）に基づいて下書きを徹底的に検査し、必要に応じて対話や修正提案を行う、インテリジェントなNostr専用投稿クライアントです。

Streamlitによる洗練されたモダンなデザインのWebインターフェースを備え、ローカルのSQLiteで全てのログを安全に管理します。

---

## 🌟 主な特徴

### 1. ⚙️ `agent.md` による独自のチェック観点定義
ユーザー自身が `agent.md` にチェックしたい観点（例：「言って」と「行って」の変換ミスが多い、柔らかいトーンにする、等）を自然言語で記述するだけで、AIがそのルールを忠実に学習・適用して下書きを検査します。

### 2. 🔍 3つのステータスによるAI判定
AIチェックを実行すると、以下の3つのいずれかに判定されます。
- ✅ **合格 (Approved)**: 問題なし。設定されたリレーへ即座に送信可能です。
- ⚠️ **要確認 (Warning)**: 誤字や不適切な表現を検知した場合。**文字単位の差分（Diff）**付きで修正案を表示し、ボタン一つで修正案を下書きに適用できます。
- 💬 **確認事項 (Question)**: 投稿内容が曖昧な場合や不足情報がある場合。AIがユーザーへ質問を投げかけ、ユーザーが回答を送信することで、その文脈を踏まえて再チェックする**対話型チェック**を行います。

### 3. 🤖 豊富なLLMプロバイダ対応
以下のLLMサービスをサポートしており、設定画面からシームレスに切り替えて利用できます。
- **商用API**: OpenAI API (デフォルト: `gpt-4o-mini`), Gemini API (デフォルト: `gemini-1.5-flash`)
- **ローカルLLM（完全オフライン）**: Ollama, LM Studio

### 4. 🔒 安全なデータ＆キー管理
- Nostrの秘密鍵 (`nsec`) やLLMのAPIキーはローカルの `.env` ファイルにのみ保存され、外部に漏洩することはありません。
- 過去のすべての投稿下書き、チェック結果、AI判定根拠、対話ログ、リレー送信結果はローカルの SQLite データベース (`prenos.db`) に保存され、いつでも振り返ることができます。

### 5. 📡 複数リレーへの一斉送信と詳細ステータス
- 設定した複数のリレーサーバー（Damus, nos.lol, yabu.me など）へ同時に一斉送信を行います。
- 送信後はリレーごとの成功・失敗状況や、`njump` などのクライアントへのリンクがすぐに表示されます。

## 🛠️ 必要要件

- **Python**: `>= 3.12`
- **環境マネージャー**: [uv](https://github.com/astral-sh/uv) (推奨)

## 🚀 クイックスタート

### 1. リポジトリの準備
プロジェクトのディレクトリに移動します。

```bash
cd prenos
```

### 2. 秘密鍵と環境変数の設定
プロジェクト直下に `.env` ファイルを作成し、Nostrの秘密鍵などを設定します。
(※ アプリケーション内の「システム設定」タブから画面上で安全に設定・更新することも可能です。)

```env
# .env
NOSTR_NSEC=nsec1... (または64桁のHEX形式の秘密キー)
OPENAI_API_KEY=your_openai_api_key (OpenAIを利用する場合)
GEMINI_API_KEY=your_gemini_api_key (Geminiを利用する場合)
```

### 3. アプリケーションの起動
`uv` を使用してアプリを起動します。依存パッケージは自動的にインストールされます。

```bash
uv run main.py
```

起動すると、自動的にブラウザが立ち上がり `http://localhost:8501` 等で Streamlit の管理画面が開きます。
(もし自動でStreamlitが立ち上がらない場合は、手動で `uv run streamlit run main.py` を実行してください。)

## 🤖 LLMの設定と切り替えガイド

本システムは、利用するLLM（言語モデル）を用途に合わせて自由に変更できます。切り替えや設定は、Streamlitの「**🛠️ システム設定**」タブから画面上で簡単に行うことができます。

### 1. OpenAI API を利用する場合
高品質なチェックや意図の正確な汲み取りを行いたい場合に適しています。

- **準備**: 
  - `.env` ファイルに `OPENAI_API_KEY=your_key` を記述するか、アプリ内「システム設定」の「OpenAI API Key」フィールドにAPIキーを入力します。
- **設定項目**:
  - **利用モデル名**: 任意のモデル名（デフォルト: `gpt-4o-mini`）
- **切り替え手順**:
  1. 「システム設定」でプロバイダーに **OpenAI API (GPT-4oなど)** を選択します。
  2. APIキーとモデル名を入力または確認し、「OpenAI設定を保存」をクリックします。

### 2. Gemini API を利用する場合
高速かつコスト効率の高いチェックを行いたい場合に適しています。

- **準備**:
  - `.env` ファイルに `GEMINI_API_KEY=your_key` を記述するか、アプリ内「システム設定」の「Gemini API Key」フィールドにAPIキーを入力します。
- **設定項目**:
  - **利用モデル名**: 任意のモデル名（デフォルト: `gemini-1.5-flash`）
- **切り替え手順**:
  1. 「システム設定」でプロバイダーに **Gemini API (Gemini 1.5/2.0など)** を選択します。
  2. APIキーとモデル名を入力または確認し、「Gemini設定を保存」をクリックします。

### 3. ローカルLLM（完全オフライン・無料）を利用する場合
外部へのデータ送信を行わず、完全にローカル環境内でプライベートにチェックを行いたい場合に適しています。

#### A. Ollama を使用する場合
1. PC上に [Ollama](https://ollama.com/) をインストールして起動しておきます。
2. 使用するモデルを事前にローカルにダウンロードします（例: コマンドラインで `ollama run qwen2.5:7b` または `ollama run llama3` 等を実行）。
3. アプリの「システム設定」でプロバイダーに **Ollama (ローカル・オフライン)** を選択します。
4. 以下の設定項目を入力し、「Ollama設定を保存」をクリックします。
   - **APIエンドポイント**: `http://localhost:11434/v1` (デフォルト)
   - **利用モデル名**: ローカルにダウンロードしたモデル名 (例: `qwen2.5:7b`, `llama3`)

#### B. LM Studio を使用する場合
1. [LM Studio](https://lmstudio.ai/) を起動し、任意の GGUF モデルをダウンロード・ロードします。
2. 左側の「Developer / Local Server」メニューから **Start Server** をクリックしてローカルAPIサーバーを起動します。
3. アプリの「システム設定」でプロバイダーに **LM Studio (ローカル・オフライン)** を選択します。
4. 以下の設定項目を入力し、「LM Studio設定を保存」をクリックします。
   - **APIエンドポイント**: `http://localhost:1234/v1` (デフォルト)
   - **利用モデル名**: ロードしているモデル名 (LM Studio画面上部に表示されるID。デフォルト: `meta-llama-3-8b-instruct`)

## 📝 使い方

1. **下書きの入力とAIチェック**:
   「新規投稿」タブで文章を入力し、`🤖 AIチェックを実行` ボタンを押します。
2. **AIとの対話・修正**:
   - 警告（Warning）が出た場合は、差分を確認して `提案を下書きに適用する` を押すか、手動で修正します。
   - 質問（Question）が出た場合は、下の回答欄に入力して `回答を送信して再検証` を押します。
3. **送信**:
   判定が Approved または Warning になると、送信セクションが出現します。最終確認を行い、`📤 Nostrに投稿する` をクリックします。
4. **チェック基準のカスタマイズ**:
   「チェック基準編集」タブから `agent.md` の記述を自由に変更し、保存できます。

## 📂 プロジェクトの構成

```plaintext
prenos/
├── doc/
│   └── spec.md          # システム仕様書
├── agent.md             # AIへのチェック観点（ユーザーがカスタマイズ可能）
├── db.py                # SQLiteデータベース操作ヘルパー
├── llm.py               # 各LLMプロバイダとの通信・プロンプト管理
├── main.py              # Streamlit Web UI アプリ本体 (兼ランチャー)
├── nostr_client.py      # Nostr-SDK を用いたイベント作成・リレー送信処理
├── pyproject.toml       # プロジェクト依存関係の定義 (uv用)
├── README.md            # 本ファイル
└── .env                 # APIキー・秘密鍵保存用ファイル (Git管理外)
```

## 🛡️ ライセンス & セキュリティに関して

- 秘密鍵 (`nsec`) や各種APIキーは、ローカルマシンの `.env` にのみ保存され、本システムが外部のサーバーに鍵情報を収集・送信することはありません。
- `.env` および SQLiteデータベース `prenos.db` は、意図しないコミットを防ぐため `.gitignore` に登録されています。
